# ML/DL Basics — Intuition Only

## 1. What Training Means
Model guesses → checks how wrong (**loss**) → nudges its weights to be less wrong → repeat millions of times.

**Gradient descent** = walking downhill blindfolded: feel which way slopes down (gradient), take a small step (update weights), repeat until you reach the bottom (lowest loss = best model).

**Simple words:** model keeps guessing, sees how wrong it was, slightly corrects itself — over and over — like feeling your way down a hill in the dark.

**Interview answer:** "Training adjusts a model's weights to minimize a loss function via gradient descent — computing the gradient of loss with respect to weights, then updating weights in the downhill direction, repeated over many examples until the model converges."

## 2. Neural Network In One Picture
`inputs → weighted sum (+ bias) → activation function → output`

Each neuron: multiplies inputs by weights, adds them (+bias), passes result through an activation function (adds non-linearity). Stack many layers → "deep" learning.

**Simple words:** multiply inputs by importance, add up, filter through activation, pass to next layer.

**Interview answer:** "A neural network layer computes a weighted sum of inputs plus bias, then applies a non-linear activation function. Stacking many layers lets the network learn complex, non-linear patterns."

## 3. What A Transformer Is (Attention)
A word's vector alone is fixed (e.g. "tower"), but **attention** lets the model adjust meaning using context — Eiffel Tower vs control tower vs tower of cards.

Every word looks at every other word in the sentence and asks "how relevant are you to me?" then blends in the important ones. Comparison is done via dot product (Query vs Key), same as vector similarity.

**Simple words:** attention = each word checking the whole sentence and pulling in context from the words that matter most, instead of relying on a fixed meaning alone.

**Interview answer:** "Self-attention lets each token compare itself against every other token via Query-Key dot products, then blend in their Values based on relevance. This gives the model context-aware meaning instead of a fixed word embedding, and captures long-range dependencies better than sequential models like RNNs."
