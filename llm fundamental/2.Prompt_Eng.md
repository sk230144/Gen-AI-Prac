# Prompt Engineering

## 1. Zero-shot vs Few-shot
**Zero-shot**: just give the instruction, no examples.
```
"Classify this review as positive or negative: 'The food was cold.'"
```
**Few-shot**: show 2-3 example input→output pairs before the real task.
```
"Amazing food!" → Positive
"Terrible service." → Negative
"The food was cold." → ?
```

**When few-shot wins:** unusual task, specific output format, tone/edge cases hard to describe in words — showing beats explaining. Zero-shot is fine for common, well-understood tasks.

**Simple words:** zero-shot = just ask. Few-shot = show a couple examples first, then ask.

**Interview answer:** "Zero-shot relies on the model's pretrained knowledge to interpret an instruction directly. Few-shot provides example pairs before the real task — more reliable when format/tone/edge-case behavior is hard to fully specify in words alone, since examples demonstrate rather than describe."

## 2. Chain-of-Thought (CoT)
Ask the model to reason step by step BEFORE the final answer, e.g. "let's think step by step."

**Why it works:** generation is autoregressive — each token conditions on everything written so far, including the model's own prior output. Writing reasoning steps first gives it a "scratchpad" to build on, instead of jumping straight to an answer on a hard multi-step problem.

```
Without CoT: "45 mph" (wrong, confident)
With CoT: "Speed = distance/time = 60/1.5 = 40 mph" (correct)
```

**Reasoning models** (trained to internally reason before answering) already do this automatically — manual "think step by step" prompting becomes redundant for them.

**Simple words:** CoT = ask the model to "show its work" first — it uses its own written reasoning as extra context, instead of guessing the answer in one shot.

**Interview answer:** "Chain-of-thought prompting asks the model to generate intermediate reasoning before the final answer. Since generation is autoregressive, reasoning steps give the model more context to build on, improving accuracy on multi-step problems. Reasoning models often make explicit CoT unnecessary since they perform extended internal reasoning automatically."
